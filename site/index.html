Why Coco

GitHub Copilot is the backend. Your agents still need a clean local path to it.

Reduce provider friction

Different coding agents expect different providers and config shapes. Coco gives them one local gateway instead of separate ad hoc setups.

Keep the boundary clear

Coco binds to localhost, proxies requests to GitHub Copilot, and avoids turning into a hosted service or a remote control plane.

Undo changes cleanly

Agent configuration is designed to be reversible. Coco can unconfigure supported agents and restore their previous state.

What It Does

Translate the compatibility layer, not your workflow.

Familiar protocol surface

Coco exposes Anthropic compatible and OpenAI compatible local endpoints so supported tools can point at one place.

Background service controls

Start, stop, restart, inspect status, and optionally install Coco as an OS login service.

Agent-aware configuration

Configure and unconfigure supported agents from the CLI while keeping configuration state explicit.

Current built-in agent support

  • Claude Code

Coco is intentionally narrow: it stays focused on being a calm, local gateway rather than becoming a chat client or an agent runtime of its own.

Install

The recommended path should be obvious.

Most users should install with npm. Deno, direct binary, and source-based setup remain available when they fit your environment.

Recommended

npm

Use this if you want the shortest path on a machine with Node.js 18+.

npm install -g coco
  1. Install Coco globally.
  2. Run coco start and complete GitHub authentication.
  3. Run coco configure claude-code or another supported agent.

Deno / JSR

Useful when Deno is already part of your toolchain.

deno install -A -g jsr:@myty/coco

Direct binary

Download a platform build from GitHub Releases when you do not want a runtime dependency.

Open releases

From source

Good for contributors or if you want to inspect the code before installing.

git clone https://github.com/myty/coco.git cd coco deno task install

How It Works

A local request path with a narrow, visible role.

Coding agent

Claude Code, Cline, or Codex sends a request to a local endpoint.

→

Coco

Coco runs on 127.0.0.1, exposes compatible endpoints, and translates request and response shapes.

  • POST /v1/messages
  • POST /v1/chat/completions
  • GET /v1/models
  • GET /health
→

GitHub Copilot

Coco authenticates with GitHub and forwards the translated request to Copilot-backed models.

Trust boundary: Coco is a local gateway, not a hosted relay. It binds to localhost and keeps configuration changes reversible.

FAQ

Common questions before you decide to install.

Do I need a GitHub Copilot subscription?

Yes. Coco routes requests through GitHub Copilot, so you need an active Copilot-backed account.

Does Coco expose an API on my network?

No. The server configuration binds to 127.0.0.1, which keeps the service local to your machine.

Can I undo a configuration change?

Yes. Coco supports unconfiguring supported agents and is designed around reversible configuration management.

Which agents are supported right now?

The current built-in registry includes Claude Code, Cline, and Codex.

Do I need Deno to use Coco?

No. npm and direct binary installs do not require Deno. Deno is only needed for the JSR and source-based paths.
